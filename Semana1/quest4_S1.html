¿Qué aspectos debe considerar un QA al validar código fuente?

Al validar un código fuente un QA debe considerar varios aspectos críticos para asegurar que el software sea funcional.

•Funcionalidad: Verificar que el código cumple con los requisitos, se deben realizar pruebas unitarias y manejar de manera correcta las excepciones y casos límite.

•Calidad del código: Se debe asegurar la legibilidad, modularidad y consistencia.

•Rendimiento: Evaluar la eficiencia y el consumo de recursos, optimizando el código cuando sea necesario.

•Seguridad: Es necesario revisar la validación de entradas, autenticación, autorización y manejo seguro de errores.

•Mantenibilidad: Comprobar que el código sea fácil de modificar, con las dependencias necesarias y actualizadas.

•Compatibilidad: Se debe asegurar que el código funcione en diferentes plataformas, navegadores y versiones de software

•Documentación: El código debe estar bien documentado, es decir, deben haber manuales y guías técnicas del funcionamiento de este.

•Cumplimiento de estándares: Es importante cumplir con los estándares técnicos de calidad y regulatorios.

¿Qué formas existen para validar el código fuente?

Actualmente existen multiples formas de validar un código fuente, las cuales se detallaran a continuación.

•Code Review: Otros desarrolladores revisan el código ya sea mediante Peer Review, Pull Requests o Pair Programming, de esta manera aseguramos calidad y buenas practicas.

•Unit Testing: Se realizan pruebas automatizadas para validar funciones individuales del código

•Análisis estático de código: Se realiza un análisis del código sin ejecutarlo, buscando errores de sintaxis, problemas de estilo y potenciales vulnerabilidades, para este análisis utilizamos herramientas como SonarQube y Codacy.

•Integration Testing: Se valida cómo interactúan los diferentes módulos del software entre sí, asegurando que funcionen de manera correcta al combinarlos

•Pruebas Funcionales: Se evalúa el código desde la perspectiva del usuario final, comprobando que las funcionalidades cumplan con los requisitos especificados.

•Continuous Integration/Continuous Deployment: En un entorno CI/CD cada vez que se sube nuevo código al repositorio, se ejecutan automáticamente una serie de pruebas que incluyen Unit Testing, Integration Testing y análisis estático del código. Algunas de las plataformas utilizadas para CI/CD son Jenkins, CircleCI y TravisCI.
