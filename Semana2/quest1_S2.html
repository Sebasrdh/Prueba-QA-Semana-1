¿Qué tipos de pruebas es posible aplicar durante el proceso de desarrollo y ejecución de un software?

Durante el proceso de desarrollo y ejecución de un software podemos aplicar multiples tipos de pruebas (Funcionales y No Funcionales), para así asegurar que el software cumpla con los requisitos y funcione correctamente en diversos escenarios.

Pruebas Funcionales

Pruebas Unitarias

Verifican el correcto funcionamiento de componentes individuales, como funciones o métodos, en aislamiento, utilizando dobles de prueba (stubs o mocks) para simular dependencias. Sirven de base para aplicaciones más complejas.

Pruebas de integración

Evalúan cómo interactúan entre sí los módulos del sistema, asegurándose de que funcionen correctamente juntos, aunque cada uno funcione bien por separado. Son previas a las pruebas de sistema.

Pruebas del sistema

Prueban el sistema completo en su entorno previsto, considerando hardware y software, para detectar errores y asegurar la calidad. Se realizan tras las pruebas de integración.

Pruebas de aceptación

Ejecutadas al final del desarrollo, verifican que el sistema cumple con los requisitos de los usuarios o partes interesadas. Involucran pruebas alfa (en el lugar de desarrollo) y beta (en el entorno del usuario).

Pruebas de Regresión

Aseguran que las actualizaciones o cambios no afecten negativamente las funcionalidades existentes. Utilizan una combinación de pruebas manuales y automatizadas para garantizar la estabilidad del sistema tras modificaciones.

Pruebas No Funcionales

Pruebas de Seguridad

Determinan la capacidad del sistema para proteger datos y evitar accesos no autorizados, evaluando autenticación, autorización y vulnerabilidades.

Pruebas de Rendimiento

Miden la eficiencia y solidez del sistema bajo diversas cargas y condiciones, incluyendo pruebas de carga, estrés y escalabilidad.

Pruebas de Usabilidad

Evalúan la facilidad de uso del sistema para el usuario final, asegurando que sea amigable y cumpla con las necesidades del usuario, mejorando su experiencia.

Pruebas de Compatibilidad

Aseguran que el software funcione correctamente en distintos sistemas operativos, navegadores y dispositivos, identificando problemas específicos de cada plataforma para ampliar el alcance del sistema.
